This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## Project Overview

Bay Area Street Hockey (BASH) league stats website. Displays scores, standings, player stats, game details, and live draft boards. Data is synced daily from Sportability (external API) into a Neon Postgres database.

## Commands

- `pnpm dev` ‚Äî Start dev server
- `pnpm build` ‚Äî Production build
- `pnpm lint` ‚Äî Run ESLint
- `npx tsx scripts/seed.ts` ‚Äî Seed database from Sportability API

## Tech Stack

- **Framework:** Next.js 16 (App Router) with React 19
- **Database:** Neon Postgres via Drizzle ORM (`drizzle-orm/neon-http`) + `rawSql()` helper for complex queries
- **Styling:** Tailwind CSS v4 with oklch() color variables
- **UI:** shadcn/ui (Radix UI primitives) in `components/ui/`
- **Client Data:** SWR hooks with server-side fallback data
- **Icons:** lucide-react

## Architecture

### Data Flow

1. **Sync:** Daily cron (`vercel.json`) hits `POST /api/bash/sync` which scrapes Sportability HTML pages and upserts game/player data into Postgres
2. **Server:** Pages are async Server Components that fetch data via functions in `lib/fetch-*.ts` (which call API routes or query DB directly)
3. **Client:** Interactive components use SWR hooks from `lib/hockey-data.ts` with `fallbackData` from server-side prefetch. Hooks auto-refresh on intervals (60-120s)
4. **Draft:** Admin creates/configures drafts via wizard ‚Üí transitions to live ‚Üí picks entered through admin board ‚Üí public board polls `/api/bash/draft/[season]` via SWR (3s interval)

### Database Constraints

**No transaction support.** The `drizzle-orm/neon-http` driver uses stateless HTTP requests ‚Äî `db.transaction()` is **not supported** and will fail at runtime. All multi-step writes use sequential `await db.` calls. Design mutations to be idempotent so partial failures are recoverable by re-running the operation. Do not introduce `db.transaction()` or `tx.` patterns without first migrating to `drizzle-orm/neon-serverless` (WebSocket driver).

### Page Pattern

Each page follows: async Server Component (data fetch + `generateMetadata`) ‚Üí Suspense boundary ‚Üí Client Component for interactivity. Pages use ISR with `revalidate = 30-60`.

### Key Directories

- `app/api/bash/` ‚Äî REST API routes with Cache-Control headers
- `app/api/bash/admin/seasons/[id]/draft/` ‚Äî Draft management API (22 endpoints)
- `lib/fetch-*.ts` ‚Äî Server-side data fetching functions (raw SQL queries)
- `lib/hockey-data.ts` ‚Äî SWR hooks for client-side data
- `lib/db/schema.ts` ‚Äî Drizzle ORM schema (38 tables)
- `lib/draft-helpers.ts` ‚Äî Snake/linear pick slot generation
- `lib/draft-trade-resolver.ts` ‚Äî Chain trade resolution engine
- `lib/schedule-utils.ts` ‚Äî Round-robin and playoff bracket generation (pure functions)
- `components/` ‚Äî Page-level components (scores-tab, standings-tab, stats-tab, etc.)
- `components/admin/draft-*.tsx` ‚Äî Draft wizard, board, and pool import components
- `components/admin/playoff-wizard.tsx` ‚Äî Playoff bracket schedule wizard
- `components/public-draft-board.tsx` ‚Äî Real-time public draft spectator view
- `scripts/` ‚Äî DB seeding and data maintenance utilities

### Standings Computation

Standings are computed from the games array (not stored): W=3pts, OTW=2pts, OTL=1pt, L=0pts. Sorted by points ‚Üí goal diff ‚Üí goals for. Regular season only (playoff games filtered out).

### URL Structure

- `/` ‚Äî Home (scores grouped by date/week)
- `/register` ‚Äî Player registration flow (temporary ‚Äî will be removed)
- `/standings`, `/stats` ‚Äî List pages filtered by `?season=YYYY-YYYY`
- `/player/[slug]`, `/team/[slug]`, `/game/[id]` ‚Äî Detail pages
- `/draft/[season]` ‚Äî Public draft board (e.g., `/draft/2026-summer`)
- `/scorekeeper/[id]` ‚Äî Live game scorekeeper
- `/admin/seasons/[id]/draft/[draftId]/board` ‚Äî Admin live draft board
- `/admin/registration` ‚Äî Registration period management
- `/admin/franchises` ‚Äî Franchise manager
- Player slugs generated by `lib/player-slug.ts` (lowercase, special chars removed)

- To run scripts that need DB access: `export $(cat .env.local | grep -v '^#' | xargs) && npx tsx scripts/your-script.ts`
